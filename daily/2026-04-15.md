## 中午（12:00 - 18:00）

### 做的事

**健康度公式的演进**

这是今天最技术密集的部分。健康度公式从最初的简单版本：

```python
# 最初版
health = 0.5 + success_rate * 0.3 + progress * 0.2
```

演进到了最终版：

```python
# 最终版
health = 0.4 + success_score(0~0.25) + progress_score(0~0.2) 
        + momentum_score(0/0.05/0.1) - fail_penalty(0/0.15) + completion(0/0.1)
```

**修过的 Bug**

1. `_is_success()` 的否定检测：`(未|无|没有|尚未|还未)` + 关键词，避免"没有完成"被误判为成功
2. `calc_health()` 失败检测：原来会错误地把中性条目（"ClawHub 未登录"）当成失败，改为只认明确的失败关键词
3. `learn_pattern()` 未定义变量：`state["driver"]` → `driver["patterns"]`
4. diversity 熵公式：平方和 → 正确的香农熵 `-Σ(p·log₂(p))`
5. `ok` 误匹配：用词边界正则 `(?<![a-zA-Z])ok(?![a-zA-Z])` 避免 ClawHub 匹配 "ok"

**状态文件的合并**

原来两个状态文件：
- `current_state.json`
- `self_driver.json`

合并为一个，加了 `_dirty` 脏标记、`_prune_log` 自动裁剪、`_dedup_projects` 去重。

**v1.0.2 的发布**

今天的最后一个技术工作。把所有修复打包，发出 v1.0.2。
